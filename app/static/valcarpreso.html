Suggested POC Architecture

  • Relational Mode
  • Separate MDM Oracle instance for EBX5 and supporting tables
  • Assume GoldenGate replicate from MDM instance to ATSE Distrubutor Oracle instance
  • No workflow
  • Triggers should be made either at GG level as exit routing or EBX5 trigger
  • INT/CERT/PROD - each has an EBX5 instance
  • Data is not moved. This should be done in tool

Who does what?

MDM Roles

  • Data Architecture
  • Development
  • Data Owner / Stewards

Data Architectural Responsibilites

  • Owns model and delivery of a model XSD file to development team for importing into EBX5
  • Data Arch should have it's own EBX5 instance strictly for development.
  • Design and implementation assistance of Data Spaces and Data Sets for use in EBX5
  • Creation / design of workflow (when implemented)

Development Team

  • Owns installation/physical server configuration for DEV/INT/CERT/PROD
  • Publish of models to Data spaces (involves automatic schema change for Rel mode)
  • Trigger development (Java)
  • GoldenGate replication
  • Any Webservices that are created from WSDLs in EBX5

Data Stewards / Owners

  • Maintenance of data content for each environment (INT/CERT/PROD
